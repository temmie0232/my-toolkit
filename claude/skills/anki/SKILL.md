# Anki Card Generator

教材を読み込み、AnkiにインポートできるTSVファイルを生成する。

## 教材フォーマット

共通フォーマット規約準拠の教材を前提とする。
規約: `~/.claude/skills/shared/references/study-material-format.md`

規約外の教材でも、見出し階層やテーブル構造から推測して対応する。
大型教材は `sessions/index.md` を入口とする分割形式のことがある（index.md の目次から辿る）。

## 起動

- `/anki` -- 教材一覧 → 選択 → カード生成
- `/anki <パス1> <パス2> ...` -- 指定ファイルから直接生成

## 手順

### Step 1: 教材の選択

`~/learning/` 配下を走査し、Markdownファイルを一覧表示する。

```
カード生成対象を選んでください（番号で指定、複数可）:

AZ900/
  1. AZ900.md（参考書）
  2. AZ900_模擬試験1.md
  3. AZ900_模擬試験2.md
  ...

ENG/瞬間英作文/
  7. 瞬間英作文.md
```

番号または名前で選択させる。複数選択可（例: 「2, 3, 4」）。

### Step 2: 教材タイプの自動判別

ファイルの内容から種類を判別する:

| 判別基準 | 概念系（→ MCQ） | ドリル系（→ Basic） |
|---------|-----------------|-------------------|
| テーブル構造 | Must Memorize表、比較表 | 対訳ドリル表（日本語 \| 英語） |
| テキスト特性 | 長い定義・解説文 | 短い対応ペア |
| 問題形式 | 選択肢付きMCQ | なし |

判別結果をユーザーに確認する（例:「AZ900.md → 概念系（MCQ）で生成します。OK？」）。

### Step 3: セクション単位で段階的にカード生成

教材が大きい場合（複数セッションがある場合）、セクション（`#` 見出し）ごとに処理する:

1. ファイル全体の見出し構造を先に把握
2. 1セクションずつ読み込み → カード生成 → 蓄積
3. 進捗を報告する（「Session 3 のカード生成中... (3/10)」）

### Step 4: TSV出力

全カードを1つのTSVファイルにまとめて出力する。

- 出力先: `~/learning/anki/`（なければ作成）
- ファイル名: `<教材ディレクトリ名>_<YYYY-MM-DD>.tsv`
- 複数ディレクトリの教材を混ぜた場合: `<dir1>_<dir2>_<YYYY-MM-DD>.tsv`

---

## TSVフォーマット

タブ区切り、3カラム。ヘッダ行なし。

```
<表面>\t<裏面>\t<タグ>
```

- フィールド内の改行は `<br>` （HTMLタグ）
- 太字は `<b>`、その他HTMLタグで整形可
- フィールド内にリテラルなタブ・改行を入れない
- タグは**スペース区切り**で複数指定

### タグ構成

```
<教材名>::<セッション>::<トピック> <教材名>::source::<種別>
```

**内容階層タグ** — カードの分類:
```
AZ900::Session1::クラウドの基本概念
AZ900::Session6::ID管理
瞬間英作文::Session1::be動詞
```

**ソース種別タグ** — 出典:
- `source::textbook` — 参考書の本文・表から生成
- `source::mock-exam` — 模擬試験をそのまま変換
- `source::generated` — 教材内容をもとに新規作成した問題

過去問チャレンジの問題は、見出しの出典表記で source タグを決める（規約で必須化済み）:
`（出典: 模擬試験N 問M）` → mock-exam ／ `（出典: 原本 <章>）` → textbook ／
`（出典: オリジナル）` → generated。出典表記が無い古い教材では文脈から推測する。

模擬試験ファイルから直接カード化するとき、教材ディレクトリに `analysis.md` の
「模擬試験 → セッション対応表」があれば、内容階層タグを `<教材名>::SessionN::<トピック>`
として付ける（対応表が無ければ `<教材名>::模擬試験N` 止まりでよい）。

## カード生成: 概念系（MCQ）

概念系教材では、**1概念あたり約5パターン**の角度からカードを作る。
全カード4択（A〜D）。正解の位置はカードごとにばらけさせる。

### 5つの出題パターン

**パターン1: 定義問い（用語 → 説明）**
用語を提示し、正しい説明を選ばせる。Must Memorize表の各行が主なソース。

表面:
```
<b>弾力性（Elasticity）の説明として正しいものは？</b><br><br>
A. リソースを手動で拡張・縮小する能力<br>
B. 需要に応じて自動的にリソースを増減する能力<br>
C. 障害発生時にサービスを継続する能力<br>
D. 災害後にシステムを復旧する能力
```

裏面:
```
<b>正解: B</b><br><br>
弾力性は需要の変動に<b>自動で</b>リアルタイム対応する能力。<br><br>
<b>他の選択肢:</b><br>
A: スケーラビリティの説明（手動含む）<br>
C: フォールトトレランス<br>
D: ディザスタリカバリ
```

**裏面のルール:** 全MCQカードの裏面は、正解の説明に加えて、**全ての誤選択肢について「なぜ間違いか」を1行ずつ必ず記載する**。省略しない。これにより、正解だけでなく「なぜ他が違うのか」を毎回確認でき、混同しやすい概念の区別が定着する。

裏面のフォーマット:
```
<b>正解: X</b><br><br>
正解の説明文。<br><br>
<b>他の選択肢:</b><br>
A: なぜAが間違いか（1行で簡潔に）<br>
B: なぜBが間違いか<br>
D: なぜDが間違いか
```

**パターン2: 逆引き問い（説明 → 用語）**
説明文や特徴を提示し、該当する用語を選ばせる。

```
<b>「需要に応じて自動的にリソースを増減する能力」を何と呼ぶ？</b><br><br>
A. 高可用性<br>B. 弾力性<br>C. スケーラビリティ<br>D. 機敏性
```

**パターン3: 比較・区別問い**
混同しやすい概念の違いを問う。比較表やペア概念から生成。

```
<b>弾力性とスケーラビリティの違いとして正しいものは？</b><br><br>
A. 弾力性は手動、スケーラビリティは自動<br>
B. 弾力性は自動、スケーラビリティは手動を含む<br>
C. 両方とも自動で動作が同じ<br>
D. 弾力性はコスト、スケーラビリティは性能に関する概念
```

**パターン4: シナリオ問い**
具体的な場面を提示し、適切な概念・サービスを選ばせる。

```
<b>ECサイトで、セール時にアクセスが急増し、終了後は元に戻る。この需要変動に対応する特性は？</b><br><br>
A. フォールトトレランス<br>B. ディザスタリカバリ<br>
C. 弾力性<br>D. 高可用性
```

**パターン5: 属性・詳細問い**
概念の特定の属性や特徴を問う。

```
<b>IaaS（Infrastructure as a Service）でユーザーの責任範囲に含まれるものは？</b><br><br>
A. 物理ホストの管理<br>B. OS・ミドルウェアの管理<br>
C. 物理ネットワークの管理<br>D. データセンターのセキュリティ
```

### ディストラクター（誤選択肢）の原則

- **同じセクション・同じカテゴリ**の概念から選ぶ。無関係な選択肢は使わない
- **もっともらしいが明確に誤り**であること
- 教材内で「紛らわしい」「混同注意」と示されている概念ペアを優先的にディストラクターに使う
- 教材に載っていない概念をディストラクターに混ぜない

### 模擬試験からのカード変換

模擬試験の問題はそのままカードに変換する:
- 表面: 問題文 + 選択肢（原文のまま）
- 裏面: 正解 + 解説（原文のまま）+ 各誤選択肢の理由

**はい/いいえ形式の問題:** 2択の問題でも、MCQ形式（4択）に変換する。元の2択を含めつつ、関連する誤選択肢を2つ追加して4択にする。

**重複排除:**
- 問題文と選択肢が実質同一の問題 → 1つだけ残す
- 同じ概念を問うが問い方・選択肢が異なる → 両方残す（異なる角度として有効）

### カード生成のソース優先順位

1. **Must Memorize表** → 定義問い、逆引き問い
2. **比較表** → 比較・区別問い
3. **本文の具体例** → シナリオ問い
4. **概念の属性説明** → 属性・詳細問い
5. **模擬試験問題** → そのまま変換
6. **過去問チャレンジ（セクション末尾）** → そのまま変換

## カード生成: ドリル/単語系（Basic）

対訳テーブルの各行から1枚のBasicカードを生成する。

表面: 日本語
裏面: 英語（またはターゲット言語）

```
表面: これはあなたのノートですか。
裏面: Is this your notebook?
```

文法ノートや注意点がある場合は、裏面の英文の下に補足として追加する:

```
裏面: Is this your notebook?<br><br><small>be動詞の疑問文: be動詞を主語の前に</small>
```

## 生成後の報告

カード生成完了時に以下を表示する:

1. **生成枚数**（概念系MCQ / ドリル系Basic / 模擬試験変換 の内訳）
2. **出力ファイルパス**
3. **タグ一覧**（ユニークなタグをリスト表示）
4. **インポート手順**:

```
Ankiへのインポート:
1. Anki → ファイル → インポート
2. 生成されたTSVファイルを選択
3. フィールドセパレータ: タブ
4. フィールド1 = Front、フィールド2 = Back
5. タグフィールド（最後のカラム）を有効化
6. ノートタイプ: 基本（裏表）
7. インポート実行
```

---

## ルール

- 教材の事実を改変しない。カード化のための言い換え・再構成はOKだが、内容の正確性を維持する
- 模擬試験の正解・解説は原文を正確に保持する
- ディストラクターは教材内の概念のみ使う。教材にない情報を混ぜない
- 出力先 `~/learning/anki/` がなければ作成する
- 大きな教材はセクション単位で処理し、進捗を都度報告する
