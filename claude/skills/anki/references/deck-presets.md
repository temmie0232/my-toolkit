# Ankiデッキオプション: 推奨プリセット

カード生成後に、この教材へ割り当てるべき **デッキオプションのプリセットと具体値** を提示するための仕様。

**核となる考え方:** プリセットを分ける基準は「カードの種類」ではなく **「締め切りの有無」**。
FSRS が1枚ごとの難易度を自動で学習して復習間隔を決めるので、人間がプリセットで分ける意味があるのは
**FSRS が吸収できない“方針”だけ** ＝ `復習間隔の上限`（締切）と `目標正答率`（取りこぼしのコスト）。
それ以外は全カード共通でよい。→ **プリセットは2つ**。

- **プリセットA＝一生モノ**: 締切なし。ゆるく長く（例: 趣味の英単語・瞬間英作文）
- **プリセットB＝締切あり**: 試験日までに仕上げる（例: 資格の概念・Cloze・模試）

---

## プリセットの選び方（生成後に必ずこの順で決める）

1. 下の「デフォルトの寄せ先」で当たりをつける
2. **ユーザーに締切を確認する**:
   「この教材に試験日（締切）はありますか？ あれば日付を教えてください。無ければ“一生モノ”扱い（プリセットA）にします」
3. **締切の有無が最終決定。ジャンルより優先する。**
   （例: TOEIC が来月ある英単語帳は、単語帳でも B）

| デフォルトの寄せ先 | カードのジャンル |
|---|---|
| A（一生モノ） | 単語帳・対訳ドリル（瞬間英作文）など、締切のない語彙・産出練習 |
| B（締切あり） | 概念系参考書・Cloze・模試/過去問・問題集・論述。試験日に向けて仕上げるもの |

- **締切なし → A**（ジャンル不問）
- **締切あり → B**（ジャンル不問）
- 1回の生成にジャンルが混在していても、**割り当て先のデッキに締切があるか**で1つに決める

## 共通設定（A・B どちらも同じ値。プリセットを新規作成する初回だけ設定すればよい）

| 設定 | 値 | なぜ |
|---|---|---|
| 復習枚数の上限 | 9999 | 復習を翌日に持ち越すと FSRS の計算が狂う。復習は必ず全部消化する前提 |
| 復習上限に関係なく新規を導入 | OFF | 復習が溜まってる日に新規まで足すと破綻する。ブレーキ |
| 上限をメインデッキから適用 | ON | 親デッキの上限で全体をコントロールでき、管理がラク |
| 習得ステップ | `1m 10m` | 新カードは「1分後→10分後」に確認して定着。それ以降は FSRS にお任せ |
| 配置順序 | 追加した順 | 教材の章の順どおりに学べる |
| 再習得ステップ | `10m` | 間違えたカードは10分後に1回だけ確認すれば十分 |
| 新規カードを集める順序 | デッキ順 | 章の順どおり |
| 集めた新規カードを並べる順序 | 集めた順 | 同上。兄弟カードは下の非表示設定が別日に散らす |
| 新規カード表示のタイミング | 復習の後 | 「忘れかけの復習」が最優先。新規は余力でやるもの |
| 日をまたいだ習得中カードの表示 | 復習と混ぜる（デフォルト） | 特に変える理由がない |
| 復習カードを並べる順序 | 想起可能率の降順 | 「今ならまだ思い出せる確率」が低い＝忘れる寸前のカードから先に救出 |
| FSRS | ON | 前提 |
| FSRSパラメータ | 月1回「最適化」ボタンを押すだけ | 回答履歴から計算精度が上がる |
| パラメータ変更時に既存スケジュールも変更 | OFF | ON にするとその日の復習が一気に膨らむ（使うのは後述の1場面だけ） |
| 信頼性チェック | OFF | 処理が遅いだけで普段は不要 |
| シミュレータ類 | 触らない | 実験的機能。遊ぶとき以外は不要 |
| 兄弟カードの非表示（新規・復習・習得中） | 3つとも ON | 双方向カード・Cloze の複数穴で「さっき答え見た」を防ぐ |
| 音声 | デフォルト | 音声カード未使用なら影響なし |
| カードの自動送り | すべて OFF（0秒） | 勝手に次へ進む機能。模試カードで誤操作事故のもと |
| 履歴欠落期間の推定正答率 / 指定日より前の復習を無視 / カスタムスケジューリング | 触らない | サボり補正・上級者向け。今は不要 |

## プリセットで変える設定（ここが本体）

| 設定 | A：一生モノ | B：締切あり |
|---|---|---|
| **新規/日** | **20** | **逆算式**（下記）。例: 未導入900枚・残り120日 → 12枚/日 |
| **目標正答率** | **0.85** | **0.90**（試験1ヶ月前に **0.92** へ） |
| **復習間隔の上限** | **36500日**（デフォルト＝無制限） | **残り日数 ÷ 2**。月1で更新（下記） |
| **苦戦カード（leech）判定** | 8回で **非表示（保留）** | 8回で **タグのみ** |
| **タイマー表示** | OFF／最長回答 **60秒** | **ON**／最長 **90秒**／解答表示でカウント停止 ON |
| **負担軽減日** | 土日を「少なめ」にしてもよい | 全曜日「標準」のまま |
| **デッキ個別の追加設定** | なし | 模試/過去問デッキだけ「このデッキ」で新規 10/日・復習 50/日 に絞る |

### 値の理由
- **新規/日**: A は数千枚の長期戦なので「続けられるペース」が正義。B は下の逆算式で「試験前の最後の3割を復習専念にできる」ペースにする。
- **目標正答率**: 0.90→0.85 に下げると復習量がガクッと減るのに、忘れる量はほんの少ししか増えない（A のコスパ最強帯）。B は本番の取りこぼしコストが高いので高め、直前はさらに上げて記憶を固める。
- **復習間隔の上限**: A は締切がないので縛る理由ゼロ。B は「どのカードも遅くとも試験までの折り返しで必ずもう1回出る」保証。
- **leech**: A は何度も間違える単語＝カードの作りが悪い→捨てて作り直す方が速い。B は試験範囲を捨てられないのでタグを目印に問題文を書き直す。
- **タイマー**: B は模試1問 20〜40秒の時間感覚を体に入れる。90秒は計測が途中で切れない保険。
- **模試デッキの個別上限**: 1枚30秒×50枚 ≒ 25分で頭打ちにするため。

### B の逆算（生成枚数と試験日から数値を埋める）
実行時の「今日の日付」と、ユーザーに聞いた試験日から計算する。

```
残り日数        = 試験日 − 今日
新規/日         = ceil( 未導入の新規カード枚数 ÷ (残り日数 × 0.7) )
復習間隔の上限   = floor( 残り日数 ÷ 2 )  日
```

- **未導入の新規カード枚数** = 今回生成して新たに学習を始めるカード枚数（新規デッキなら生成した全枚数）。
- 新規/日 の `×0.7` は「最後の3割の期間は新規ゼロで復習に専念する」ためのマージン。
- 復習間隔の上限は **月1回、残り日数の変化に合わせて更新する**
  （例: 120日前→60日、90日前→45日、30日前→14日、14日前→7日）。
- 模試/過去問デッキは、逆算した新規/日と `10/日` の **小さい方** を「このデッキ」個別上限にする。

---

## 「既存スケジュールも変更」を ON にする唯一の場面

B で復習間隔の上限を **絞ったとき**（例: 60→45日）だけ、一度だけこのチェックを入れて保存する。
遠い未来へ飛んでいたカードが手前に引き戻される。その日の復習は一時的に増えるが織り込み済み。
**終わったらチェックは外す。**

## 出力フォーマット（`生成後の報告` に必ず含める）

カード生成の報告の最後に、次のブロックを埋めて提示する。

```
## 推奨Ankiデッキ設定

判定: プリセット<A/B>（<一生モノ / 締切あり>）
  根拠: <締切の有無 ＋（Bなら）試験日 YYYY-MM-DD・残り N日>

▼ このプリセットに設定する値
  - 新規/日: <A=20 / B=逆算結果>（<Bなら計算式を1行>）
  - 目標正答率: <A=0.85 / B=0.90（1ヶ月前0.92）>
  - 復習間隔の上限: <A=36500 / B=floor(残り÷2)=N日・月1更新>
  - leech判定: <A=8回で保留 / B=8回でタグのみ>
  - タイマー: <A=OFF・60秒 / B=ON・90秒・解答で停止>
  （Bかつ模試デッキ: このデッキ個別に 新規<min(逆算,10)>/日・復習50/日）

▼ 共通ベース（プリセットを初回に作るときだけ設定・詳細は deck-presets.md）
  FSRS ON / 習得1m 10m / 再習得10m / 兄弟3つON / 復習は想起可能率の降順 …

適用: デッキオプション → プリセット<A/B>を選択（無ければ作成）→ 上の値を入力 → 保存
```

- 締切が **不明なら必ず聞く**。聞かずに A/B を決めない。
- B で復習間隔の上限を今回新しく絞る場合は、「一度だけ『既存スケジュールも変更』を ON にして保存」も添える。
