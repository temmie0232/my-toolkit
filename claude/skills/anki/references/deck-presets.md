# Ankiデッキオプション: 推奨プリセット（正典）

カード生成後（Step 5）に提示する**デッキオプション推奨の唯一の正**。
2026-07-09 MAGI決議（5-0）で全面改訂。旧版原文は `deck-presets.source.md`（アーカイブ・参照しない）。

**核となる考え方:** プリセットを分ける基準は「カードの種類」ではなく**「締め切りの有無」**。
FSRS が1枚ごとの難易度と間隔を自動で学習するので、人間がプリセットで決めるのは
FSRS が吸収できない方針 ＝ 締切まわり（`復習間隔の上限`・`目標正答率`）だけ。→ プリセットは2つ。

- **プリセットA＝一生モノ**: 締切なし。ゆるく長く（例: 英単語・瞬間英作文）
- **プリセットB＝締切あり**: 試験日までに仕上げる（例: 資格の概念・Cloze・模試）

## Step 5 実行手順（この順で・毎回）

1. **このファイルを最初から最後まで読み直す**（記憶で再構成しない）
2. **確認質問を1ターンに束ねて聞く**:
   - 締切の有無（文脈から試験日の候補が分かるときは「◯◯試験 YYYY-MM-DD でよいですか？」と
     Yes/No形式で、なければ日付を質問。**聞かずに決めない**）
   - 既存デッキへの追加なら、デッキに残っている未導入（新規のまま）の枚数
   - （任意）1日にAnkiへ使える時間・負担を軽くしたい曜日
3. **Part 2** の式とガードで可変値を計算する
4. **Part 1** を**逐語転載**して出力する（サマリ層 → 38項目本表 → 検算行）
5. B なら **Part 3** の運用カレンダーを続けて出力する（行文言は固定・日付のみ計算）
6. 「選択肢の意味も知りたい」と言われたら**対照表**を提示する

## Part 2: 可変値の決め方

### 判定

1. 締切を確認（Step 5 手順2の形式で。**聞かずに決めない**）
2. 締切なし → **A** ／ 締切あり → **B**（**ジャンルより優先**。単語帳でも試験が近ければ B）

### 逆算（B）

```
残り日数        = 試験日 − 今日
未導入枚数      = 既存デッキの新規残（デッキ一覧の新規数、または検索 deck:対象 is:new の件数）＋ 今回生成枚数
新規/日         = ceil( 未導入枚数 ÷ (残り日数 × 0.7) )   ← 最後の3割の期間は新規0で復習に専念
復習間隔の上限   = floor( 残り日数 ÷ 2 )
```

### ガード（提示前に必ず通す）

- **負荷予告（必須）**: 定常時の復習 ≈ 新規/日 × 10枚（公式目安）。分換算は recall≈10秒/枚・
  逐語MCQ≈30秒/枚で教材の内訳に応じて加重。「導入初期はこれより少ない」と必ず添える
- **過大チェック**: 予測負荷が可処分時間（聞けている場合）を超える、または可処分時間が不明で
  新規/日が30枚を超えるときは、数値をそのまま提示せず「カードを間引く／教材を絞る／模試カードを
  優先する」の対話に切り替える
- **残り45日以下**: 2回のタッチを1回に統合 — 初回から 上限=floor(残り÷2)・目標正答率0.92・
  reschedule 1回。D-30イベントは行わない
- **残り14日未満**: 新規は最初の数日で導入を打ち切り、フィルターデッキ中心の直前運用を提案する
- **模試/過去問サブデッキを使っている場合のみ**: そのデッキの「このデッキ」個別上限で
  新規 min(逆算値, 10)/日 に絞ってよい。**復習の個別上限は設定しない**（復習上限に達すると
  新規導入も止まり、未消化が翌日に繰り越されるため）。「上限をメインデッキから適用 ON」の下では、
  この個別上限は親デッキから学習するときに意図どおり効く

## 対照表（全38項目の意味と全選択肢 — 通常は出力しない。求められたら提示する）

**1. 一日の新規カード導入枚数の上限**（数値）
その日に新しく学習を始めるカードの上限。新規1枚は定常的に約10枚/日の復習を生む（公式目安「新規は復習上限の1/10以下」）。A=20は継続優先、B=逆算は「試験前に導入し終える」最小ペース。

**2. 一日の復習枚数の上限**（数値）
その日に表示する復習の上限。達すると残りの復習が隠れ、既定では新規導入も止まる。**9999**=事実上無制限にして毎日全消化。なおFSRSは復習が遅れても計算が壊れることはない — 消化できない日があっても、忘れる分の損だけで翌日以降普通に回せる（サボり＝破損ではない）。

**3. 復習枚数の上限に関係なく新規カードを導入する**（ON/OFF）
ONだと復習上限に達した日も新規を入れ続ける。**OFF** — 遅れているのに新規を積むと未消化が悪化する（公式も「追いつくまで新規を止めよ」）。

**4. 上限をメインデッキから適用**（ON/OFF)
サブデッキから直接学習するときも親デッキの上限を効かせる。**ON** — 親で総量を管理でき、模試サブデッキ等を作っても全体が崩れない。

**5. 習得ステップ**（テキスト）
新規カードが「復習カード」になる前に同日内で確認する間隔の列。FSRS有効時は「1日以上のステップ非推奨」「ステップ数は最小限（同日の複数反復は長期記憶にほぼ寄与しない）」（Ankiマニュアル）、「単一の10m/15m/20m/30mが妥当。複数の短ステップも非推奨」（FSRSチュートリアル）。`1m 10m` はAnki出荷時の既定値だが、上記により **10m 単独**。若いカードの正答率が目標より大きく低いときだけ 2h 等の追加を検討。

**6. 配置順序**（追加順／ランダム）
新規カードのキュー内の並び。**追加順**=教材の章の順で学べる。ランダムは順序の癖を消したいとき用。

**7. 再習得ステップ**（テキスト）
復習で「もう一度」にしたカードを同日内で確認する間隔。**10m** 単独 — 以降の間隔はFSRSが再計算する。

**8. 苦戦カードと判定する、復習での忘却回数**（数値）
「もう一度」の累計がこの回数に達すると苦戦カード（leech）判定。**8**=Anki既定。以後は半分ごと（12,16…）に再警告。

**9. 苦戦カードへの処置**（カードを休止／タグを付けるだけ）
判定時の自動処置。休止=デッキから外れて出なくなる／タグ=印を付けて出し続ける。**A=カードを休止**（作りの悪いカードに時間を吸われない。休止カードは月1回見直して書き直す）。**B=タグを付けるだけ**（試験範囲は捨てられない。タグを目印に問題文を直す。現行Ankiの既定もタグのみ）。

**10. 新規カードを集める順序**（デッキの並び順(上から)／デッキの並び順(上から)→ランダム(ノート単位)／位置(新規カード番号)の昇順／位置(新規カード番号)の降順／ランダム(ノート単位)／ランダム(カード単位)）
その日の新規枠に「どのカードを選ぶか」。**デッキの並び順(上から)**=章の順どおり。位置系は新規カード番号順、ランダム系は分野を混ぜたいとき。

**11. 集めた新規カードを並べる順序**（カードタイプ順／集めたときの順序／カードタイプ順→ランダム／ランダム(ノート単位)→カードタイプ順／ランダム(カード単位)）
選んだ新規を「どの順で見せるか」。**集めたときの順序**=10の順のまま。カードタイプ順系は表裏カードを固めたいとき用（⑥の非表示設定があるので通常不要）。

**12. 新規カード表示のタイミング**（復習と混ぜる／復習の後に表示／復習の前に表示）
新規と復習のどちらを先にやるか。**復習の後に表示** — 忘れかけの救出が最優先。疲れて途中でやめた日に犠牲になるのが新規側で済む。

**13. 日をまたいだステップの習得中（再習得中）カード表示のタイミング**（復習と混ぜる／復習の後に表示／復習の前に表示）
ステップが翌日以降にまたがる習得中カードの表示位置。**復習と混ぜる**=既定のまま（10m単独ステップならほぼ発生しない）。

**14. 復習カードを並べる順序**（期日超過日数が多い順→同日数はランダム／期日超過日数が多い順→同日数はデッキの並び順／デッキ並び順→デッキ内は期日超過日数多い順／復習間隔が短い順／復習間隔が長い順／難度が低い順／難度が高い順／正答可能性が低い順／正答可能性が高い順／期日超過の割合が大きい順／ランダム(カード単位)／追加した順／追加日時が新しい順(「追加した順」の逆順)）
その日の復習の表示順。**期日超過日数が多い順→同日数はランダム**=Anki既定で、遅れが無い〜小さいうちの公式推奨。**正答可能性が高い順**=未消化が溜まったときの公式推奨（24.11リリースノート: シミュレーションによりバックログ時はこちらが良い・将来の既定候補）。「正答可能性が低い順」は一見良さそうだが、溜まったときに選ぶと救えるカードまで失う（シミュレーションで劣後）。毎日全消化できている限り、並び順は結果にほぼ影響しない。

**15. 目標正答率**（数値 0.70〜0.99）
復習が来た時点で覚えていてほしい確率＝FSRSの中心ダイヤル。高いほど間隔が縮み負荷が増える（0.90超は負荷急増・0.97超は圧倒的・UIも0.70〜0.97の外で警告）。**A=0.85**（0.90→0.85で負荷が大きく減るのに忘却はわずかしか増えない）。**B=0.90 → 試験30日前に0.92**（本番の取りこぼしコストが高い）。

**16〜18. 兄弟カードの非表示**（新規／復習／習得中(日またぎ)、各ON/OFF）
同じノート由来のカード（双方向・複数穴Cloze）を同じ日に出さない。**3つともON** — 「さっき裏で答えを見た」直後の水増し正解を防ぐ。

**19. 音声を自動再生しない**（ON/OFF）
カード表示時の音声自動再生を止めるか。**OFF**（=自動再生する・既定のまま）。音声つきカード未使用なら影響なし。

**20. 解答側を表示後の「音声を再生」アクションでは、質問側を再生しない**（ON/OFF）
「音声を再生」操作で質問側の音声を省くか。**OFF**（既定のまま）。

**21. 最長回答秒数**（数値）
統計に記録する1枚あたり回答時間の上限（超過分は切り捨て）。**A=60／B=90** — 模試MCQは1問20〜40秒かかるため、計測が頭打ちにならない保険。

**22. 回答タイマーを表示する**（ON/OFF）
画面に経過秒を出すか。**A=OFF**（急かされる感覚は不要）／**B=ON**（本番の時間感覚を体に入れる）。

**23. 解答を表示したときに、タイマーのカウントを停止する**（ON/OFF）
解答面で秒を止めるか。**A=OFF／B=ON** — 計測対象を「考えている時間」に限定する。

**24. 質問表示時間（秒）**（数値）
自動送り（Auto Advance）で質問面を表示しておく秒数。**0**=自動送り無効。

**25. 解答表示時間（秒）**（数値）
同・解答面。**0**=無効。

**26. 音声再生終了を待つ**（ON/OFF）
自動送りが音声の終了を待つか。**OFF**（自動送り自体を使わない）。

**27. 質問表示時間経過後の自動アクション**（解答を表示／リマインダーを表示）
自動送り発火時の動作。**解答を表示**（既定・0秒なので発火しない）。

**28. 解答表示時間経過後の自動アクション**（カードを今日は非表示／回答 (もう一度)／回答 (正解)／回答 (難しい)／リマインダーを表示）
同・解答面後の動作。**カードを今日は非表示**（既定・0秒なので発火しない）。誤って「回答 (正解)」等にすると勝手に採点されるため触らない。

**29〜35. 負担軽減日**（月〜日、各: 最小限／少なめ／標準）
曜日ごとに復習量を減らす重み（標準=1.0／少なめ=0.5／最小限=0）。**全曜日「標準」が基本** — 締切ありで減らすのは自分の首を絞める。どうしても重い曜日だけ「少なめ」に（全曜日を下げると相対配分のため全部標準と同じ。すでに付いている期日には遡及しない）。

**36. 復習間隔の上限（日）**（数値）
次回復習までの間隔の最大値。**A=36500**（事実上無制限。締切がないので縛る理由なし）。**B=floor(残り日数÷2)** — どのカードも遅くとも折り返しでもう1回出る＝試験前に最低1周の保証。180日未満でAnkiが「非推奨」警告を出すが、締切用途では想定内。触るのは計2回だけ（Part 3）。

**37. 履歴欠落期間の推定正答率**（数値）
復習履歴が欠けている期間についてFSRSが仮定する当時の正答率。**0.90**（既定のまま） — 「38」を使った・履歴を消した・他アプリから移行した場合だけ調整する。

**38. 指定日より前の復習を無視**（日付）
指定日以前の復習ログをパラメータ最適化から除外する。**空欄のまま** — 採点の付け方を根本的に変えた場合などに使う上級者向け。履歴自体は消えない。

## 設計の根拠（出典レベルつき・改訂時の検証注記）

- **習得ステップ 10m 単独**: 〔公式〕Ankiマニュアル「1日以上のステップ非推奨・ステップ数は最小限・同日反復は長期記憶にほぼ寄与しない」＋〔準公式〕FSRSチュートリアル「単一の10m/15m/20m/30mが妥当・複数短ステップも非推奨」。※「複数短ステップ非推奨」の出典はチュートリアル側のみ
- **復習ソートの条件分岐**: 平常=「期日超過日数が多い順→同日数はランダム」〔公式既定〕／未消化滞留時=「正答可能性が高い順」〔公式・24.11リリースノート。バックログ限定の推奨〕
- **間隔上限の2タッチ運用**: FSRSは「次回期日=想起率が目標正答率まで落ちる日」で間隔を組むため、毎日全消化している限り、期日が試験後に飛んだカードも試験日時点の想起率≥目標が構造的に成り立つ（fuzz・負荷分散で最大0.5pp程度緩む近似保証）。D-30の1タッチは、安定度の過大推定（模試MCQの選択肢位置を覚えた等）による「隠れ忘却」を試験9日前までに検出する保険。月次の絞り込みは「rescheduleなしでは遠い期日のカードに届かず、rescheduleありでは副作用の反復」のため廃止〔決議・2026-07-09〕
- **復習上限9999**: 上限で隠すと取りこぼし増＋新規停止。FSRSは遅延復習で壊れない〔公式FAQ整合〕。「9999」「上限=残り÷2」自体は公式明文のないコミュニティ運用知〔運用知・Expertium提案ほか〕
- **leech**: 閾値8〔公式既定〕。処置の既定も現行Ankiはタグのみ（マニュアルの「休止する」記載は旧仕様）
