# 論述・記述式の最適化手順

年度別過去問（問題冊子・解答例/出題趣旨・採点講評）を素材に、答案を書く力に最適化した
対策教材を作る。対象は2サブタイプ:

- **論文式**: 高度試験の午後II（ST/SA/PM等）。2時間で数千字の論述
- **記述式**: AP午後・高度試験の午後I。読解して数十字の記述で答える

## 基本方針

**答案を代筆しない。「型」と「採点者の目」を抽出する。**
論述・記述式の価値は自分の経験・知識を設問の型に流し込む練習にあり、optimizer の仕事は
(a) 出題の型と頻出テーマの分析 (b) 採点講評からの減点パターン類型化 (c) 想起で回せる
粒度の演習（一問一答）作り。答案を書く練習そのものは原本の問題冊子を直接使う。教材はその足場。

## 入力と出力

```
~/learning/<試験名>/
├── PM2/<年度>/qs.md, ans.md, cmnt.md   ← 原本（変更禁止）。年度別の問題・解答例・採点講評
├── <試験名>_論文対策.md 等              ← このジャンルの出力（対策教材）
└── analysis.md                          ← 出題分析
```

- 原本の年度別アーカイブは**一切変更しない**
- 解答例・採点講評が無い年度も扱える（その年度は出題分析のソースにのみ使う）

## analysis.md（出題分析）

全年度分を**実際に数えて**作る（感覚で書かない）:

- 年度 × 問番号 × テーマ × 設問構造の表
- テーマ系統の頻度集計（例: DX戦略 / 新規事業 / リスク管理 / 組織・人材）
  → セッション配分と「ネタの仕込み」の優先順位の根拠になる

## セッションの骨格

```markdown
<a id="session-1"></a>

# Session 1: 設問の型と答案の構造

## このセッションの全体像

論文試験は「経験の暗記」ではなく「型への流し込み」。設問ア（概要・800字）→
イ（施策・1600字）→ ウ（評価・1200字）の定型を先に固めると、どのテーマが来ても
書き出しで迷わなくなる。Session 2 以降のテーマ別対策は全てこの型の上に載る。

## これだけ覚えろ（Must Memorize）

| 設問 | 型（書くべき要素と順序） |
|------|------------------------|
| 設問ア | 事業概要 → 課題 → 構想の位置づけ（800字以内） |

## 減点パターン

| 減点の類型 | 採点講評の指摘 | 回避策 |
|-----------|---------------|--------|
| 概要と本文の不整合 | 「本文と整合性がとれないものが散見」（出典: R6春 講評） | 設問アで書いた課題をイ・ウで必ず参照する |

## 型の解説

（字数配分・時間配分・段落構成。原本の出題趣旨・講評から導ける範囲で）

## 過去問チャレンジ

### 問題 1（出典: オリジナル）
**設問アの定型構成は？**

**答え: 事業概要 → 課題 → 構想の位置づけ（800字以内）**

---

# Session 2〜: テーマ系統別（analysis.md の頻度順）

## このセッションの全体像
## これだけ覚えろ（Must Memorize）   ← そのテーマで問われる観点・キーワード
## 出題例と設問分解（出典: <年度> 問N）
## 過去問チャレンジ                   ← 一問一答（下記）
```

## 各ブロックの作り方

### 減点パターン（このジャンルの核）

- 採点講評（cmnt）から「〜が散見された」「〜してほしい」を抽出し、類型 × 指摘 × 回避策の
  表にする。**出典年度を必ず付ける**
- 複数年度で繰り返される指摘を最優先に置く（毎年言われる = 確実に採点基準）
- 講評が無い試験では省略してよい（創作しない）

### Must Memorize

- 論文式: 設問の型 × 書くべき要素。テーマ別セッションでは「そのテーマの定番観点」
  （例: DX案件なら「経営戦略との整合 / 既存業務への影響 / 効果測定」）を解答例から抽出
- 記述式: 設問タイプ（抜き出し / 理由記述 / 計算 / 提案）× 解答セオリーの表。
  模範解答のキーワードを優先して抽出する

### 過去問チャレンジ（一問一答のみ）

- 答案全文を書かせる課題は置かない（原本で演習する）。**想起で答えられる粒度**に分解する:
  「設問アの定型構成は？」「R5秋 問1 設問イで問われた観点は？」「この減点を避けるには？」
- 全て一問一答形式（規約参照）。出典（年度・問・設問）を必ず付ける
- 実際の年度演習への導線を末尾に書く（「原本 PM2/2024/qs.md 問1 を2時間で書く」等）

### 記述式（AP午後等）の場合の差分

- セッション単位は「分野」（情報セキュリティ / データベース等の選択分野）
- 出題例と設問分解では、本文のどこが解答根拠かを示す（読解の型）
- 解答例のキーワードを Must Memorize に集約（部分点はキーワード一致で決まりやすい）
