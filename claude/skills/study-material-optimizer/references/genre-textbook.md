# 参考書型の最適化手順

概念の解説文が主体の教材（試験対策の教科書・合格教本など）。
実績例: `~/learning/AZ900/AZ900.md`（全10セッション、tutor完走・anki 420枚の実績あり）。

## セッションの骨格

```markdown
<a id="session-6"></a>

# Session 6: IDとアクセス管理 (Identity & Access Management)

**ドメイン: Azureの管理とガバナンス（出題比率 30-35%）**   ← 試験系のみ・任意

## このセッションの全体像

認証（あなたは誰？）→ 認可（何をしていい？）の2段階で考えると全体が繋がる。
トピック1〜3が認証の道具、4〜6が認可の道具。Session 5 の「ネットワーク境界」の防御に
対し、ここは「人」の側の防御。前提: Session 1 の共同責任モデル（ID管理は常にユーザー責任）。

## これだけ覚えろ（Must Memorize）

| 用語 | 定義（試験で問われるポイント） |
|------|-------------------------------|
| 弾力性 (Elasticity) | 需要に応じて**自動的に**リソースを増減する能力 |

## 1. トピック名

**なぜ存在するか**: 〜という問題を解決するため。（1行）

（本文: 比較表・覚え方）

**混同注意**: 弾力性 vs スケーラビリティ — 「自動かどうか」が分かれ目。

### 試験Tips
- 「自動的にリソースが増減」→ **弾力性**

## 2. 次のトピック
...

## 数値まとめ            ← 数値・上限・期間が3つ以上あるセッションのみ

| 項目 | 数値 | 備考 |
|------|------|------|

## 過去問チャレンジ

### 問題 1（出典: 模擬試験2 問14）
...（4〜8問）
```

## 各ブロックの作り方

### このセッションの全体像

- そのセッションの概念群を貫く**ストーリー**を1つ選ぶ（例:「防御→検知→証明の3つの柱」
  「見る・減らす・仕組みの3段階」）。見つからない場合はトピックの依存順序を書く
- 前セッションとの接続・前提知識（「Session 1 の◯◯を使う」）を必ず1つは入れる
- 効果測定済みのパターン: 2段階/3本柱のような少数のグルーピング + 学ぶ順序の理由

### これだけ覚えろ（Must Memorize）

- 過去問がある場合: analysis.md の頻度表で上位のポイントから埋める。
  過去問で引っかかりやすい区別・対比（「自動 vs 手動」等）は太字で強調
- 表に収まらない横断ルール（「物理層は常にプロバイダー責任」等）は表の下に箇条書きで

### 本文

- 概念ごとに `##` で区切る。1トピックの説明は tutor が1回で扱える量（概念2〜3個）を意識
- **なぜ存在するか** を各トピック冒頭に1行。「何の問題を解決するか」で書く。
  学習者が最初に聞くのは大抵「なぜこれが要るのか」で、これが教材に無いと
  tutor が教材の範囲を超えた即興を強いられる
- 似た概念の対比は**比較表**が最も効果的。積極的に使う
- 紛らわしいペアには `**混同注意**:` を明示（anki のディストラクター選定ソース）
- 「覚え方」「試験Tips」「試験の罠」で出題パターンと引っ掛けを明示
- 冗長な説明は簡潔に。ただし重要な詳細（制限値・条件・例外）は省かない

### 過去問チャレンジ

問題のソース優先順位:

1. 模擬試験・過去問ファイルからそのセッション該当分を抽出（出典: 模擬試験N 問M）
2. 元教材の章末問題（出典: 原本 <章>）
3. 不足分はセッション内容からオリジナル作成（出典: オリジナル）

- 各セッション最低4問、理想は6〜8問
- 頻度の高いトピックの問題を優先。Must Memorize でカバーした区別が最低1問は問われる構成に
- **実試験由来（模試・原本）は4択を逐語保持**。解説には誤選択肢がなぜ違うかも書く
  （anki の裏面がそのまま利用する）
- **オリジナル問題は一問一答形式で作る**（規約の一問一答フォーマット参照）。
  4択にしない — 創作ディストラクターは品質リスクで、選択肢形式の練習は実試験由来が担う。
  答えは一語〜1行（「〜を何と呼ぶ？」「AとBの分かれ目は？」の形が作りやすい）

## 試験のない教材の場合（技術書・教養書など）

対策する試験が存在しない参考書もこのジャンルで扱う。差分は以下だけ:

- **試験Tips・出題比率・試験の罠は書かない**（根拠となる試験が無い）。
  代わりに「実務での使いどころ」を教材から導ける範囲で書いてよい
- **過去問チャレンジの代わりに `## 理解確認`** を置く（4〜8問、全て一問一答・出典オリジナル）。
  4択問題は一切作らない
- Must Memorize・全体像・なぜ存在するか・混同注意・数値まとめは試験の有無に関係なく必須
