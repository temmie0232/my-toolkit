# ドリル型の最適化手順

短い対応ペア（出題→答え）の反復で「わかる」を「できる」に変える教材
（瞬間英作文、単語帳、一問一答の暗記帳など）。
実績例: `~/learning/ENG/瞬間英作文/瞬間英作文.md`（13セッション・790文）。

## 基本方針

ドリル教材の価値は**ペアの量と回しやすさ**。解説を膨らませず、
「型の理解を最短で済ませて反復に入れる」構成にする。

## 教材冒頭: トレーニング方法論を保持する

原本にトレーニング方法論（サイクル法、回し方、1文あたりの秒数目安など）がある場合、
**必ず冒頭に「はじめに」として残す**。ドリル教材はやり方自体が学習効果の核心で、
これを削ると教材がただの対訳リストになる。加えて「この最適化版の使い方」
（tutor でどう使うか、答え列を隠して使う等）を短く添える。

## セッションの骨格

```markdown
<a id="session-1"></a>

# Session 1: be動詞・指示語・人称代名詞

## このセッションの全体像

英作文回路の土台。be動詞の文を肯定・疑問・否定のどれでも考え込まずに出せるようにする。
最初の関門は「疑問文での be動詞の扱い」と「主語と be動詞の一致」。

## これだけ覚えろ（Must Memorize）

| 文型 | パターン | 要注意ポイント（罠） |
|------|----------|----------------------|
| this / that | This is ~. / Is this ~? | 疑問文は be動詞を文頭に1つだけ（Is that **is** ~? ✗） |

## １ this / that

**パターン**：`This / That is ~.`（肯定）／`Is this ~?`（疑問）／`This is not ~.`（否定）

**ポイント**
- 疑問文は文頭に be動詞を**1つだけ**。`Is that is his house?` は典型ミス

### ドリル

| # | 日本語（これを見て即答） | 英訳（答え） |
|:--:|------|------|
| ① | これは良い本です。 | This is a good book. |
| ② | あれは本物の花ではない。 | That is not a real flower. |
```

## 各ブロックの作り方

### このセッションの全体像

- 2〜3行でよい。「このセッションで自動化する型は何か」「最初の関門はどこか」を書く

### これだけ覚えろ（Must Memorize）

- **型 × 罠**の表にする。定義の暗記ではなく「作るときに踏むミス」を先に頭に入れるのが目的
- 罠は原本の解説・注記から拾う（「〜としがち」「〜に注意」）

### 単元（型・文型・単語グループごとに `##`）

- **パターン** 1行 + **ポイント** 箇条書き2〜3個まで。解説はこれ以上膨らませない
- 直後に `### ドリル` の表。**ドリル表の契約**（規約で定義、anki が依存）:
  - 3列: `#`／出題／答え
  - **答え列の見出しに「答え」を含める**（例: `英訳（答え）`、`意味（答え）`）
  - 1行 = 1ペア。行内の改行は `<br>`
- 答えに一言の文法補足が要る場合は、表の下ではなく該当行の答えセル内に `<br>` で添える
  （anki が行単位でカード化するため、表外の補足はカードに乗らない）

### 過去問チャレンジは作らない

ドリル表自体が演習。4択に変換すると「見て選ぶ」練習になり、
産出（自分で出す）トレーニングという教材の目的を壊す。

## 単語帳系の場合

- 単元 = 品詞・テーマ・レベル等、原本のグルーピングに従う
- ドリル表は `| # | 単語 | 意味（答え） |`。例文がある場合は答えセル内に `<br>` で添える
- 1セッション = 50〜100語目安（文型ドリルより多めでよい）
