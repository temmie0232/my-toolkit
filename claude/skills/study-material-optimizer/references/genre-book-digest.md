# 読み物型（一般書ダイジェスト）の最適化手順

学習目標のない一般書（ビジネス書・教養書・ノンフィクション・小説）を、
短時間で内容を把握し、後から再参照できる「読みやすい資料」にまとめる。

## 基本方針

**目的は暗記ではなく把握と再参照。** 学習系ジャンルと違い、演習・Must Memorize・
学習セッション分割を作らない。代わりに3層で読める構造を作る:

1. **2分**: 「この本を一言で」＋「一枚まとめ」で骨格を思い出せる
2. **15分**: 「全体の構造」＋章ダイジェストで全体を追える
3. **必要時**: 章題が原本のままなので、深掘りしたい章だけ原本に戻れる

- 要約は圧縮であって解釈の創作ではない。**著者の主張を自分の意見・批評で上書きしない**
- 分量の目安: 原本の3〜5%、15分で読み切れる量（8,000〜15,000字）。
  超えそうなら章を削るのではなく、1章あたりの粒度を下げる
- 章立ては原本の構成を保持する（学習セッションへの再分割はしない）

## 出力の骨格

```markdown
# <書名> ダイジェスト

> 原本情報（著者・全何章）と圧縮方針（何を削ったか: 事例の詳細、繰り返される主張 等）

## この本を一言で

（中心主張を1〜3行）

## 一枚まとめ

（中心の問い → 著者の答え → 主要な根拠2〜3。10〜15行）

## 全体の構造

（章がどう繋がるか。「1〜3章で問題提起、4章で転回、5章以降は各論」のような議論の流れ）

## キーコンセプト

| 概念 | 意味 | 登場章 |
|------|------|--------|

---

# 第1章: <原本の章題のまま>

**要点**
- 著者の主張の圧縮を3〜5個
**論拠・事例**: 代表的なもの1〜2個を1行ずつ

---
（以下、章ごとに繰り返し）

## 心に残る一節

> 原文ママの引用（第N章）

## 実践ポイント（実用書のみ）

- [ ] 著者が推奨する行動のチェックリスト
```

## 各ブロックの作り方

### 一枚まとめ

- 「中心の問い → 著者の答え → 主要な根拠2〜3」の骨格で書く。
  **これだけ読めば人にこの本を説明できる**状態が合格ライン

### 全体の構造

- 章の羅列ではなく議論・物語の**流れ**を書く（参考書型の「全体像」の全書版）。
  論の転回点・パート境界がどこかを示す

### 章ダイジェスト

- 要点は著者の主張の圧縮。事例・データは代表1〜2個に絞り、残りは削る
  （何を削ったかは冒頭の圧縮方針に書く）
- 章題は原本のまま保持する（原本参照の索引を兼ねる）
- 章間の接続が強い本では **つながり**: を1行足してよい

### 心に残る一節

- 原文ママ＋出典（章）。言い換え禁止 — このブロックだけは一切圧縮しない

### 実践ポイント（実用書のみ）

- 著者が推奨する行動をチェックリスト化。著者が言っていない行動を創作しない

## 小説・物語の場合

- 一枚まとめ → **「ネタバレなし紹介（3行）」と「全あらすじ」に分け、前者を先に置く**
- キーコンセプト → 登場人物表（人物・立場・関係）
- 章ダイジェスト → 章ごとの梗概（時系列の飛び・視点の変化をメモ）
- 実践ポイントは作らない。「テーマ・モチーフ」の節を置いてよい（本文から導ける範囲）

## 検証（このジャンルの忠実性チェック）

- 「心に残る一節」の引用が原文と一字一句一致するか**全件**突合
- 一枚まとめ・章要点の主張が原本に実在するか（章ごとに1主張をサンプル突合）
- 数値・固有名詞・年号を raw と突合（最低5件）

## tutor / anki との関係

- tutor: 学習進捗管理の対象外。ただし「この本について話したい」用途で章単位に使える
- anki: 原則カード化対象外。ユーザーが明示的に望む場合のみキーコンセプト表から
  recall Basic / Cloze を生成する
