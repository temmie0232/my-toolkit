# 問題集型の最適化手順

問題＋解説の繰り返しが本文の大半を占める教材（Java SE Silver問題集、一問一答集など）。

## 基本方針

**解説文を新たに書き起こさない。問題と解説が主役のまま学習最適化する。**
問題集の価値は問題そのものにあり、optimizer が参考書風の本文を創作すると
(a) 原本に無い事実が混入するリスク (b) 問題を解く時間が減る、の二重に損をする。
optimizer の仕事は「問題群から知識の地図を抽出して前置きする」ことと「出典を保った整理」。

## セッションの骨格

```markdown
<a id="session-3"></a>

# Session 3: 演算子と分岐文

## このセッションの全体像

この章の問題は大きく「演算子の優先順位」「暗黙の型変換」「switch文の制約」の3グループ。
前半の型変換ルール（Session 2）を土台に、罠の大半は「コンパイルエラーになるか、
実行時例外か、正常動作か」の判別を突いてくる。

## これだけ覚えろ（Must Memorize）

| 頻出知識 | ポイント |
|---------|---------|
| byte+byte の結果型 | int になる（byte変数に代入するとコンパイルエラー） |

## ひっかけパターン

| 罠の類型 | 見抜き方 |
|---------|---------|
| コンパイルエラー選択肢 | 代入先の型とリテラルの型を先に確認する |
| 「実行時例外」と「コンパイルエラー」の混同 | 型チェックはコンパイル時、null参照は実行時 |

## 過去問チャレンジ

### 問題 1（出典: 原本 3章 問12）
**次のコードをコンパイル・実行した結果はどれか。**

```java
int a = 10;
System.out.println(a++ + ++a);
```

- A. 20
- B. 21
- C. 22
- D. コンパイルエラー

**正解: C**
> a++ は評価後に加算（10を返す）、++a は加算後に評価（12を返す）。10 + 12 = 22。
> A: インクリメントを両方無視した場合の値
> B: a++ を ++a と同じと誤解した場合の値
> D: この式は文法的に正しい
```

## 各ブロックの作り方

### このセッションの全体像

- その章の問題群を**罠のグループ**で括る（「この章は◯◯の判別を突いてくる」）。
  問題を解く前に何を試されるか分かっていると、学習が受け身にならない
- 前セッションの知識への依存があれば明示する

### これだけ覚えろ（Must Memorize）

- 章の解説群から**繰り返し登場する知識**を抽出して表にする。問題を解く前の武装
- 数値・制限・仕様の暗記項目（配列の既定値、アクセス修飾子の範囲等）を優先

### ひっかけパターン

- その章の問題が仕掛ける罠を類型化した表。「罠の類型 × 見抜き方」の2列
- 原本の解説で「〜に注意」「誤りやすい」と書かれている箇所が主なソース

### 過去問チャレンジ（セッションの主体）

- 原本の問題を出典付きで配置する。**コード・実行結果・選択肢は逐語保持**
  （インデントとコードブロック化のみ可）。コード問題は1文字の違いが正解を変える
- 原本の解説は圧縮・再構成OK。ただし正解と技術的事実は改変禁止
- 解説に誤選択肢の理由が無い場合、章の解説から導ける範囲で補う（導けなければ書かない）
- 問題数が多い章は、全体像で括った罠グループ順に並べ替えてよい（原番号は出典で追える）
- 補足問題をオリジナルで足す場合は一問一答形式で（規約参照）。4択のオリジナル問題は作らない

## 解説の薄い過去問集の場合（問題と正解だけの教材）

Webからまとめた過去問集など、解説がほぼ無く章分けも無い教材（例: 世界遺産検定の過去問142問）。
問題集型の亜種として次の手順で扱う:

- **トピック再クラスタリング**: 無分類の問題群を内容でグルーピングしてセッションを設計する
  （例: 地域別・時代別・制度系）。元の問題番号は出典表記で保持する（出典: 原本 問N）
- **Must Memorize は答えから逆構成する**: 解説が無いので、正解が示す事実そのものを
  知識表に起こす（「カトマンズの谷の被災 → ネパール大地震(2015)」）。
  正解から直接導けない背景知識は足さない（教材外知識の混入禁止の原則どおり）
- **ひっかけパターンは作れる範囲でだけ**: 選択肢の構成から罠が読み取れる場合のみ
  （例: 同地域の類似遺産を並べる）。読み取れなければこの表自体を省略してよい
- **画像依存問題の隔離**: 選択肢が画像ラベル（a/b/c/d）のみで、画像なしでは
  成立しない問題は、通常セッションに混ぜず末尾の `## 画像参照問題（原本参照）` に
  一覧で隔離する。tutor / anki の対象外であることを冒頭に明記する

## 章が大きい場合

- 1章あたりの問題が20問を超える場合はセッションを分割する（1セッション10〜15問目安）
- 分割時は「ひっかけパターン」を分割後の各セッションに対応する分だけ再配分する
