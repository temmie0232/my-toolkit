# クイズ・判定・カルテ更新の規則

測定の質がこのスキルの生命線。ここが甘いとカルテが実力とズレて、適応そのものが壊れる。

## 1. 先手試行の3択の作り方

- 選択肢はすべて日本語の「大意」で書く（文法用語を使わない）
- 正解1つ + もっともらしい誤読2つ。**露骨なハズレを作らない**——3つとも「その文を雑に読んだらそう読めてしまう」もっともらしさを持たせる。当てずっぽうで切れる選択肢は診断力を持たない
- 誤答肢の優先順位:
  1. **カルテのメモにある過去の誤読傾向**（例:「whichの説明先を取り違えがち」→ それを再現する誤読を仕込む）
  2. 日本人学習者の典型誤読パターン:
     - 修飾のかかり先の取り違え（長い主語・後置修飾）
     - 時制・相の無視（完了を現在として読む、仮定法を直説法として読む）
     - 指示語（it/they/which）の指す先の取り違え
     - 省略の見落とし（関係代名詞の省略、共通部分の省略）
     - 否定の範囲の取り違え（部分否定を全否定に）
     - 知っている単語の意味への引きずられ（多義語の直訳）
- 正解の位置（A/B/C）は毎回変える

## 2. ZPDゲート（試行を出すかどうかの判定）

急所文に含まれる概念のうち、カルテで箱0〜1のものを数える:
- **0〜2個** → 試行を出す（ちょうどいい背伸び）
- **3個以上** → 試行を出さない。「この文は今日は一緒に読もう」と言って解説に直行する。当てずっぽうのテストは何も測れないうえに、不快感だけが残る

「わからない」と答えられた場合も同じ扱い（責めずに解説へ）。ただしカルテには誤答として記録せず、関連概念のメモに「未挑戦」と残すだけにする（わからないと言える判断はむしろ正確な自己認識）。

## 3. 双子文（たしかめ）の作り方

目的: 「その文を覚えたか」ではなく「その**文法**が取れるようになったか」の転移を測る。

- **同じ文法の骨組み** + **対象文法だけが効いている**文を新作する
- 語彙は Tier 1 レベルの基本語だけ（語彙でつまずかせない——測りたいのは文法だけ。語彙ノイズが混ざると何を測ったのか分からなくなる）
- 基本語で足りる範囲で、ことば帳（data/words.md）の既出語を優先して混ぜてよい——別枠の単語クイズを新設せずに語彙の再会機会を稼ぐ（義務化された単語テストより、例文の中での自然な再会のほうが続く）
- 長さは原文と同じか短く
- トピックはユーザーの興味・その日の話題に寄せてよい（例文が自分事だと想起が強くなる）
- 直近の試行の成績が良すぎる（9割超）→ 双子文を一段応用形に。低すぎる（5割未満）→ 一段やさしく、3択形式に落とす（「ほぼ勝てるが時々惜しい」= 85%前後が学習効率と気持ちよさの両立点）

## 4. 判定基準（内部・厳格）

- **正答**: 大意が構造レベルで合っている。語彙の小さな取り違えだけなら正答扱いにしてメモに残す
- **誤答**: 構造の取り違え（かかり先・時制・主述・指示語・否定範囲のミス）
- **判定に迷う**: 箱を動かさない。メモだけ更新。無理に白黒つけない（誤判定はノイズよりたちが悪い）
- 甘い判定・お世辞の「正解！」は禁止。判定を緩めることはユーザーの学習機会を静かに奪う行為
- 表示ルール（×を出さない・部分肯定から入る）は explain-style.md 側。**内部判定を表示のために曲げない**

## 5. カルテ更新（SKILL.md の規則の補足）

- 箱の意味: 0=出会った / 1=学習中 / 2=芽が出た / 3=定着 / 4=卒業（正式用語で呼べる）
- 昇格は1セッション1概念につき+1まで。箱3→4だけは「別の日の正答2回」が必要
- 誤答での降格は-1まで（一気に0に戻さない。1回のミスは事故かもしれない）
- 試行（3択）と双子文の両方に同じ概念が関わった場合、その日の判定は**双子文を優先**（新作文への転移のほうが強い証拠）
- リベンジマッチの正誤も同じ規則でカルテに反映する

## 6. bank.md（リベンジ文庫）の運用

- 追加するのは「試行または双子文で誤読した文」の原文。長い文は誤読箇所を含む1文だけ切り出す（前後の文脈が要る場合は出典メモに一言）
- 期日: 追加時=明日 → 成功ごとに 3日 → 7日 → 14日後
- **連続成功2回で「克服」**: 行を削除し、karte.md ヘッダの克服カウンタを+1。ユーザーには「この文はもう卒業」と一言
- 失敗したら連続成功を0に戻し、期日を明日に
- 上限20文。超えたら最も古い行を削除（義務の山を作らない——復習キューの負債化はこの種のツールの典型的な死因）
- リベンジマッチは1セッション最大1問。期日超過が溜まっていても1問だけ（溜まった分を消化させようとしない）
