# 教材Markdownフォーマット規約

pdf2text → study-material-optimizer → tutor / anki のパイプラインで共通利用する契約。
この規約が「最適化済み教材がどう見えるか」の唯一の定義。optimizer はこれに準拠して出力し、
tutor / anki はこれを前提にパースする。規約からの逸脱は下流スキルを壊す。

## ジャンル

教材は以下3ジャンルのいずれかとして最適化する。共通ブロックは全ジャンル同じで、
本文・演習の構成だけがジャンルで変わる。

| ジャンル | 例 | 教材の性質 | 演習の形式 | ankiカード |
|---------|-----|-----------|-----------|-----------|
| **参考書型** | AZ-900教科書、AP合格教本 | 概念の解説文が主体 | 4択（過去問チャレンジ） | recall Basic／Cloze 中心 |
| **問題集型** | Java SE Silver問題集 | 問題＋解説が主体 | 原本の問題を出典付きで配置 | 逐語MCQ＋recall Basic/Cloze |
| **ドリル型** | 瞬間英作文、単語帳 | 短い対応ペア（出題→答え）の反復 | ドリル表 | Basic表裏 |

anki のカード形式は知識のタイプで決まる（定義・逆引き・分かれ目・シナリオ=recall Basic、
数値・列挙=Cloze、MCQは模試・過去問の逐語変換と答えを一意にできないシナリオのみ）。
詳細は anki スキル側で定義する。

**対象外（当面）**: 論述式試験の過去問（記述・論文）、年度別過去問PDFの単純アーカイブ。
これらは最適化せずそのまま保管する。

## ディレクトリ規約

全教材は `~/learning/` 配下にサブディレクトリで格納する。

```
~/learning/
├── tutor-index.md              ← 全教材の進捗一覧（tutor が管理）
└── <教材名>/
    ├── raw.md                  ← pdf2text 出力の原本（変更禁止）
    ├── <教材名>.md             ← 最適化済み教材（〜15セッション目安）
    ├── sessions/               ← 大型教材のみ（単一ファイルの代わり。どちらか一方）
    │   ├── index.md            ← 入口: 教材概要・目次・Part構成
    │   └── NN-<タイトル>.md     ← 1ファイル = 1セッション
    ├── 模擬試験N.md 等          ← 模擬試験・過去問（あれば、複数可・原本扱い）
    ├── analysis.md             ← optimizer の出題分析（模試がある場合）
    └── tutor-progress.md       ← tutor が生成する学習進捗
```

- 15セッション相当を超える教材は `sessions/` 分割を推奨（生成の再開性と読み込み効率のため）。
  分割時も各セッションファイルの中身は通常のセッション形式（1ファイルに `# Session N:` 1つ）
- 模擬試験ファイルの命名は自由（既存例: `AZ900_模擬試験1.md`）。原本扱いで変更禁止
- 既存教材に raw.md が無いケースは許容するが、新規教材では raw.md に統一する

## 教材ファイルの全体構造

```markdown
# 教材タイトル

> 教材の概要と最適化の前提（何を分析してどう重み付けしたか）

---

## 目次

**Part 1: <まとまり名>**          ← セッション10超のとき推奨（例: 中1レベル / 午前対策）
- [Session 1: セクション名](#session-1)
- [Session 2: セクション名](#session-2)
...

---

<a id="session-1"></a>

# Session 1: セクション名
（セッション本体）

---

<a id="session-2"></a>
...
```

## 見出しルール（変更禁止の契約）

| レベル | 用途 |
|--------|------|
| `#` | **セッション単位**（tutorの1セッション）。`# Session N: タイトル` 形式。最初の `#` のみ教材タイトル |
| `##` | セッション内のトピック・機能ブロック |
| `###` | トピック内の小見出し・個々の問題 |

- `#` セッション間は `---`（水平線）で区切る
- 各セッション直前にアンカー `<a id="session-N"></a>` を配置（目次ジャンプ用）

## セッションの構成

### 共通ブロック（全ジャンル・この順序で冒頭に置く）

#### 1. ドメイン行（任意・試験系のみ）

`# Session N:` の直下:

```markdown
**ドメイン: クラウドの概念（出題比率 25-30%）**
```

#### 2. このセッションの全体像（必須）

```markdown
## このセッションの全体像

認証（あなたは誰？）→ 認可（何をしていい？）の2段階で考えると全体が繋がる。
トピック1〜3が認証の道具、4〜6が認可の道具。Session 5 が「ネットワーク境界」の防御
だったのに対し、ここは「人」の側の防御を扱う。前提: Session 1 の共同責任モデル。
```

- 2〜6行。トピックの列挙ではなく**流れ・つながり**を書く: どの順で学ぶか、なぜその順か、
  前セッション・前提知識とどう接続するか
- なぜ必須か: 概念がバラバラに見えると頭に入らない。tutor はセッション冒頭でこのブロックを
  使って導入する（学習者フィードバックに基づく必須ブロック）

#### 3. これだけ覚えろ（Must Memorize）（必須）

```markdown
## これだけ覚えろ（Must Memorize）

| 項目 | 覚えるべきポイント |
|------|-------------------|
| 用語A | 定義・試験で問われるポイント |
```

- セッション全体の要点を表形式で凝縮。中身はジャンルで変える:
  参考書型 = 用語×ポイント / 問題集型 = 頻出知識 / ドリル型 = 型×罠

### 本文と演習（ジャンル別）

#### 参考書型

```markdown
## 1. トピック名

**なぜ存在するか**: 手動スケールでは急な負荷に人が追いつかない、という問題を解決する。

（本文。似た概念の対比には比較表を積極的に使う）

**混同注意**: 弾力性 vs スケーラビリティ — 「自動かどうか」が分かれ目。

### 試験Tips
- 「自動的にリソースが増減」→ **弾力性**
```

- 各トピック冒頭に `**なぜ存在するか**:` 1行（強く推奨）。存在理由から入ると定着が早く、
  tutor が学習者の「なぜ？」に教材の範囲内で答えられる
- 紛らわしい概念ペアは `**混同注意**: A vs B — 一言` で明示（anki がディストラクター選定に使う）
- セッション末尾（過去問チャレンジの前）に任意で数値まとめ:

```markdown
## 数値まとめ

| 項目 | 数値 | 備考 |
|------|------|------|
| 障害ドメイン | 最大3 | 可用性セット |
```

  数値・上限・期間が3つ以上登場するセッションで置く。数値系は取りこぼしやすく、
  直前復習と anki の属性カードのソースになる
- 末尾に `## 過去問チャレンジ`（4〜8問、後述の問題ブロック形式）

#### 問題集型

解説文を新たに書き起こさず、問題と解説を主役のまま学習最適化する:

```markdown
## このセッションの全体像
## これだけ覚えろ（Must Memorize）   ← 解説群から抽出した頻出知識の表
## ひっかけパターン                   ← この章の問題が仕掛ける罠の類型表
## 過去問チャレンジ                   ← 原本の問題（出典付き）。これがセッションの主体
```

- コード・実行結果・選択肢は原本から**逐語保持**（インデント整形のみ可）
- 解説は圧縮・再構成OK。ただし正解と技術的事実は改変禁止

#### ドリル型

```markdown
## １ <型の名前>

**パターン**: `This / That is ~.`（肯定）／ `Is this ~?`（疑問）

**ポイント**
- 疑問文は be動詞を文頭に1つだけ（`Is that is his house?` ✗）

### ドリル

| # | 日本語（これを見て即答） | 英訳（答え） |
|:--:|------|------|
| ① | これは良い本です。 | This is a good book. |
```

- **ドリル表の契約**: 3列（`#`／出題／答え）で、**答え列の見出しに「答え」を含める**。
  anki はこの形状を検出して Basic カード（表=出題列、裏=答え列）を生成する
- 教材固有のトレーニング方法論（サイクル法等）が原本にある場合、冒頭に「はじめに」として
  保持する。やり方自体が学習効果の核心なので削らない
- 過去問チャレンジは不要（ドリル表自体が演習）

## 問題ブロックのフォーマット（参考書型・問題集型・模擬試験共通）

```markdown
## 過去問チャレンジ

### 問題 1（出典: 模擬試験2 問14）
**問題文**

- A. 選択肢A
- B. 選択肢B
- C. 選択肢C
- D. 選択肢D

**正解: B**
> 解説。誤選択肢がなぜ違うかにも触れる。

---

### 問題 2（出典: オリジナル）
...
```

- 選択肢は `- A.` 〜 最大 `- F.`。複数選択は問題文に「（2つ選択）」を明記し `**正解: B, E**`
- コードを含む問題はコードブロックで逐語保持
- **出典表記は必須**。値は3種:

| 出典表記 | 意味 | ankiのsourceタグ |
|---------|------|-----------------|
| `（出典: 模擬試験N 問M）` | 模試・過去問ファイルからの転載。正解・解説は原典に忠実 | `source::mock-exam` |
| `（出典: 原本 <章など>）` | 元教材に載っていた演習問題 | `source::textbook` |
| `（出典: オリジナル）` | optimizer が教材内容から新規作成した問題 | `source::generated` |

- なぜ必須か: (a) 正解の検証を原典突合でできる (b) anki が source タグを正しく付けられる
  (c) 弱点トピック別の問題抽出ができる

## analysis.md（模擬試験・過去問がある場合）

optimizer が出題分析の成果物として出力する。原本の模試ファイルには手を加えない。

```markdown
# <教材名> 出題分析

## トピック別出題頻度

| トピック | 出題数 | 対応セッション |
|---------|--------|---------------|

## 模擬試験 → セッション対応表

| 模擬試験 | 問題 | セッション | トピック |
|---------|------|-----------|---------|
| 模擬試験1 | 問1 | Session 6 | Identity Protection |
```

- 重み付けの根拠を監査可能にする。また anki が模試からカードを作るとき、この対応表があれば
  `<教材名>::SessionN::<トピック>` の内容階層タグを付けられる（なければ模試名タグ止まり）

## 機械可読マーカー一覧

tutor / anki がパースに依存するマーカーの集約。**表記を変えないこと**（括弧は全角）。

| マーカー | 使う側 | 用途 |
|---------|--------|------|
| `# Session N: <タイトル>` | tutor, anki | セッション境界・進捗単位・タグ第2階層 |
| `<a id="session-N"></a>` | 目次 | ジャンプ |
| `## このセッションの全体像` | tutor | セッション導入の提示 |
| `## これだけ覚えろ（Must Memorize）` | tutor, anki | 全体像の要点提示・定義/逆引きカードのソース |
| `**なぜ存在するか**:` | tutor | 「なぜ」質問への回答ソース |
| `**混同注意**:` | anki | ディストラクター選定・比較問題のソース |
| `## 数値まとめ` | anki | Cloze（穴埋め）カードのソース |
| `## ひっかけパターン` | tutor, anki | 罠の類型（問題集型） |
| `## 過去問チャレンジ` | tutor, anki | 実践問題・カード変換 |
| `### 問題 N（出典: ...）` | anki | source タグの判定 |
| `- A.` 〜 `- F.` ／ `**正解: X**` | tutor, anki | 選択肢・正解のパース |
| ドリル表（答え列見出しに「答え」） | anki | Basic カード生成 |
| `---`（セッション間） | tutor | 区切り |

## 不要コンテンツの基準

以下は教材最適化時に削除する:

- あとがき、著者紹介、出版情報
- 広告・宣伝
- 試験申込手順（学習内容ではない）
- 改訂履歴
- 同じ内容の重複（複数箇所に同一説明がある場合は1箇所に集約）

以下は残す:

- 序文のうち、学習の前提知識や試験の構成説明に関わる部分
- 教材固有のトレーニング方法論（ドリル型で特に重要）
- 用語集・索引（参照用に末尾に配置）
